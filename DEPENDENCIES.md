# Зависимости проекта AR-promenad

## Основные библиотеки

### 1. CameraX (версия 1.3.1)
Современная библиотека для работы с камерой Android.

**Компоненты:**
- `camera-core` - базовая функциональность
- `camera-camera2` - реализация Camera2 API
- `camera-lifecycle` - интеграция с lifecycle компонентов
- `camera-view` - UI компоненты для отображения камеры
- `camera-extensions` - расширенные возможности (ночной режим, HDR и т.д.)

**Применение:**
- Захват изображений в реальном времени
- Анализ кадров для ML модели
- Preview камеры для пользователя

---

### 2. ML Kit
Google ML Kit для машинного обучения на устройстве.

**Компоненты:**
- `object-detection` (17.0.1) - обнаружение объектов
- `image-labeling` (17.0.8) - классификация изображений
- `image-labeling-custom` (17.0.2) - кастомные модели

**Применение:**
- Базовое обнаружение объектов в кадре
- Предварительная классификация (что это: здание, памятник и т.д.)
- Быстрая обработка на устройстве без интернета

---

### 3. TensorFlow Lite (версия 2.14.0)
Легковесная версия TensorFlow для мобильных устройств.

**Компоненты:**
- `tensorflow-lite` - основная библиотека
- `tensorflow-lite-support` - вспомогательные функции
- `tensorflow-lite-metadata` - метаданные моделей
- `tensorflow-lite-gpu` - GPU ускорение

**Применение:**
- Запуск кастомной модели распознавания достопримечательностей
- Обработка изображений с камеры
- Классификация конкретных московских объектов

**Модели, которые будем использовать:**
- Custom модель для распознавания достопримечательностей Москвы
- Обучена на датасете фотографий московских локаций

---

### 4. ARCore (версия 1.41.0)
Google ARCore для дополненной реальности.

**Применение:**
- Отображение AR маркеров с информацией о достопримечательностях
- Наложение 3D объектов в реальном пространстве
- Определение плоскостей и глубины сцены
- AR навигация до объектов

**Требования:**
- Устройство должно поддерживать ARCore
- Android 7.0 (API 24) или выше
- Камера с автофокусом

---

### 5. Room Database (версия 2.6.1)
Официальная ORM библиотека от Google для SQLite.

**Применение:**
- Хранение информации о достопримечательностях
- Кеширование результатов распознавания
- История посещенных мест
- Избранные локации

**Структура БД:**
```
Landmarks: id, name, description, latitude, longitude, type, images
History: id, landmark_id, visit_date, recognition_confidence
Favorites: id, landmark_id, added_date
```

---

### 6. Google Play Services - Location (версия 21.1.0)
Сервисы геолокации от Google.

**Применение:**
- Определение текущего местоположения пользователя
- Контекстные подсказки (какие достопримечательности рядом)
- Расчет расстояния до объектов
- Геофенсинг (уведомления при приближении к достопримечательности)

---

### 7. Kotlin Coroutines (версия 1.7.3)
Асинхронное программирование для Kotlin.

**Применение:**
- Асинхронная загрузка данных из БД
- Обработка ML модели в фоновом потоке
- Работа с сетью (если будет)
- Управление lifecycle-aware операциями

---

### 8. Gson (версия 2.10.1)
Библиотека для работы с JSON.

**Применение:**
- Парсинг JSON файлов с данными о достопримечательностях
- Сериализация/десериализация моделей данных
- Импорт/экспорт настроек

---
